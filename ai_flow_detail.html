MULAI
1
User Melakukan Aksi di Aplikasi
React (Vite) · Port 5173 · http://localhost:5173

User membuka aplikasi Matcha di browser. Di sini ada 3 jenis aksi yang bisa dilakukan:

💬 Aksi 1 — Chat
Ketik pesan di kotak chat, misalnya "Saya ingin jadi Data Analyst". Dikirim ke POST /chat
📄 Aksi 2 — Upload CV/LinkedIn
Drag & drop file PDF. Dikirim ke POST /upload dengan file_type = cv atau linkedin
🔍 Aksi 3 — Analisis Job
Paste Job Description lalu klik Analisis. Dikirim ke POST /analyze-job (shortcut — skip 2 langkah AI)
React 19 + Vite 8 Axios untuk HTTP request session_id unik per user
HTTP Request ke backend
2
FastAPI Backend Menerima Request
main.py · Port 8000 · http://localhost:8000

Server FastAPI di main.py menerima permintaan dari frontend. Sebelum menjalankan AI, backend selalu memuat data lama user dari database agar tidak lupa konteks percakapan sebelumnya.

📥 Yang diterima
session_id, user_input, dan opsional agent_state dari frontend. Untuk upload: file PDF + file_type.
🗄️ Muat sesi lama
Panggil load_session(session_id) → ambil data profil, CV, riwayat intent, roadmap dari SQLite. Digabung ke state saat ini.

Jika aksinya adalah Upload PDF: file langsung diekstrak oleh PyMuPDF menjadi teks mentah, disimpan ke uploads/ (untuk preview), dan disimpan ke SQLite. Tidak perlu LLM!

📂 File disimpan di
uploads/{session_id}_{file_type}_{filename}.pdf
💾 Teks disimpan ke
session["cv_text"] atau session["linkedin_text"] di SQLite
Jalankan AI Graph
3
LangGraph Memulai Pipeline AI
agent/graph.py · matcha_graph.invoke(state)

LangGraph adalah framework yang mengatur urutan node AI seperti sebuah alur kerja. State (data) mengalir dari satu node ke node berikutnya. Semua data user (MatchaState) dibawa sepanjang alur ini.

📦 State yang dibawa
user_input, user_profile, cv_text, linkedin_text, messages, detected_intent, learning_roadmap, dll.
🔀 Cara kerjanya
Setiap node baca state → lakukan tugasnya → kembalikan state yang sudah diupdate. Node berikutnya lanjut dari state terbaru.
Mulai dari Node 1: intent_classifier Selalu urut: Node 1 → Node 2 → Router → Node pilihan
Node 1
4
Intent Classifier — "User Mau Apa?"
agent/nodes.py · intent_classifier_node() · Node 1 LangGraph

Langkah pertama AI: pahami maksud user. Node ini menentukan ke mana alur selanjutnya. Ada 2 cara mendeteksi intent, dipakai berurutan (primary → fallback):

🧠 Primary — IndoBERT Lokal
Model fine-tuned yang sudah di-training khusus untuk Matcha. Berjalan lokal di komputer, tidak perlu internet. Dipanggil lewat fungsi _classify_intent_indobert(user_input).

Model dimuat sekali saat pertama dipanggil, lalu di-cache di memory supaya request berikutnya langsung cepat. Hasilnya: nama intent + confidence score (0–1).
⚡ Fallback — Groq LLM
Jika IndoBERT tidak tersedia atau error, sistem otomatis pakai Groq API. Kirim prompt ke model LLM (misal: llama-3.3-70b), minta hasil JSON berisi intent dan confidence.

Ada auto-fallback antar model: llama-3.3-70b → qwen3-32b → llama-3.1-8b
🎯 8 Jenis Intent yang Bisa Dikenali
CAREER_EXPLORATION SKILL_INQUIRY RESOURCE_REQUEST CONSTRAINT_UPDATE PUSH_BACK CONFIRMATION CV_REVIEW LINKEDIN_REVIEW
🔄 Bonus: Drift Detection
Sistem menyimpan riwayat 3 intent terakhir. Kalau intent user tiba-tiba berubah jauh dari pola sebelumnya (misal dari CAREER_EXPLORATION ke PUSH_BACK terus), drift_detected = True dan AI akan kasih notifikasi ke user bahwa ada perubahan arah.
Node 2
5
User Profiler — "Siapa Usernya?"
agent/nodes.py · user_profiler_node() · Node 2 LangGraph

Node ini membangun atau memperbarui profil karir user dari percakapan. Tujuannya: agar AI tahu siapa user, latar belakangnya, dan mau ke mana.

⚡ Fast-Path (Hemat Token)
Kalau user_profile.target_role sudah ada di state (artinya user sudah onboarding), node ini langsung skip LLM — hemat waktu dan biaya API yang signifikan.
🤖 Pakai Groq LLM (jika perlu)
Kirim riwayat chat + input user ke LLM. LLM ekstrak info: current_role, target_role, current_skills, hours_per_week, budget_idr, timeline_months.
🔀 Merge Profil
Profil baru di-merge ke profil lama. Field yang sudah ada tidak ditimpa kecuali ada nilai baru yang lebih lengkap. Profil terus bertambah seiring percakapan.

Pilih 1 dari 4 Node
7
Agent Node — Eksekusi Tugas Utama
agent/nodes.py · Salah satu dari 4 node dijalankan
Node 3 — Skill Gap Analyzer
Intent: CAREER_EXPLORATION · SKILL_INQUIRY · RESOURCE_REQUEST
Node paling kompleks. Tugasnya: analisis kesenjangan skill user vs kebutuhan industri, dan buat learning roadmap. Cara kerjanya step by step:

1. Ambil data paralel (2 thread bersamaan):
  • ChromaDB (lokal): cari skill yang dibutuhkan untuk target role, daftar job relevan, dan kursus yang cocok
  • Tavily (web search): cari tren industri dan skill yang sedang in-demand di internet

2. Potong teks agar hemat token: CV dipotong maks 1800 karakter, LinkedIn 1000 karakter, Job Description 1500 karakter.

3. Kirim semua data ke Groq LLM dengan prompt detail. LLM menghasilkan JSON berisi: chat_response, match_rate, skill_gaps, ats_analysis, learning_roadmap.

4. Hitung ATS Match Rate dengan ML Model:
ATS ML API :8080 (primary) local import fallback ChromaDB + Tavily paralel Groq LLM JSON output

Cara kerja ATS ML Model: Kirim CV + Job Description ke POST http://localhost:8080/predict. Model Random Forest mengekstrak 4 fitur (TF-IDF similarity, evidence score, experience alignment, keyword density) lalu prediksi skor 0–100. Jika API offline, langsung import predict_ats_match_rate() dari lokal. Skor ini menimpa skor dari LLM supaya lebih akurat dan konsisten.

Node 4 — CV Reviewer
Intent: CV_REVIEW
Ambil teks CV dari state (sudah diekstrak saat upload). Kirim ke Groq LLM bersama target_role dan skill_gaps. LLM menghasilkan review detail: kelebihan CV, kekurangan, dan rekomendasi keyword ATS yang perlu ditambahkan. Output: chat_response + ats_analysis.
Groq LLM cv_text dipotong maks 2500 karakter output: cv_pros, cv_cons, suggested_keywords
Node 5 — LinkedIn Reviewer
Intent: LINKEDIN_REVIEW
Sama seperti CV Reviewer tapi untuk profil LinkedIn yang di-export sebagai PDF. LLM menganalisis headline, summary, bagian experience, dan skills. Memberikan saran optimasi agar profil lebih mudah ditemukan recruiter dan sesuai target karir.
Groq LLM linkedin_text dipotong maks 1500 karakter saran: headline, about, skills section
Node 6 — General Responder
Intent: PUSH_BACK · CONFIRMATION · CONSTRAINT_UPDATE
Node untuk percakapan umum yang tidak butuh analisis berat. Jika user menolak rekomendasi (PUSH_BACK), AI mengakui dan menawarkan alternatif. Jika user update constraint (CONSTRAINT_UPDATE), AI mencatat dan menawarkan revisi roadmap. Jika ada drift terdeteksi, AI tambahkan notifikasi perubahan arah.
Groq LLM 6 pesan terakhir sebagai konteks respons maks 3 paragraf
Simpan hasil
8
Simpan Sesi ke Database
agent/memory.py · save_session() · SQLite: matcha_sessions.db

Setelah node agent selesai, hasilnya langsung disimpan ke database SQLite supaya user tidak kehilangan progress — meskipun browser ditutup atau server restart.

💾 Data yang disimpan
user_profile, skill_gaps, detected_intent, previous_intent_history, cv_text, linkedin_text, job_description, ats_analysis, learning_roadmap, status upload file.
🔑 Cara penyimpanan
Pakai INSERT ... ON CONFLICT DO UPDATE. Artinya: jika session_id sudah ada → update, belum ada → buat baru. Tidak akan duplikat.
Return response ke frontend
9
Kirim Jawaban ke Frontend
FastAPI Response → React · Render di layar user

Backend mengembalikan JSON berisi response (teks jawaban AI) dan agent_state (data lengkap). React menerimanya dan langsung menampilkan hasilnya ke user.

💬 Chat Response
Teks jawaban AI muncul di kotak chat. Mendukung markdown untuk formatting yang rapi.
📊 ATS Dashboard
Match rate gauge, daftar mastered skills, skill gaps, suggested keywords ATS dirender ke panel analisis.
🗺️ Learning Roadmap
Fase-fase belajar dengan jadwal, kursus berbayar & gratis, durasi per phase ditampilkan sebagai timeline interaktif.
